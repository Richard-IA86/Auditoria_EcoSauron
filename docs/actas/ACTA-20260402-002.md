## 1. Incidencia Reportada

* **Repositorio:** `planif_pose`

* **Módulo:** `src/normalizador/transformer.py` -> `detectar_duplicados()`

* **Síntoma:** Al detectar duplicados *de proceso* (filas funcionalmente idénticas con el mismo `_ID_INGESTA`), la
  función los eliminaba valiéndose de un chequeo lógico `isin([ids_a_eliminar])`. Dado que los índices conservados y
  eliminados compartían el mismo ID, la directiva `~df["_ID_INGESTA"].isin()` eliminaba **ambas** ocurrencias en lugar
  de solo purgar la repetida, incurriendo en pérdida silenciosa de datos válidos.

## 2. Resolución y Parche (Hotfix)

1. **Modificación del Motor Lógico:** Se sustituyó el rastreo eliminatorio basado en el campo `_ID_INGESTA`. Ahora
   captura el "índice literal Pandas (`.index`)" del grupo duplicado exceptuando el primero, y se aplica una purga
   directa mediante `df.drop(index=indices_a_eliminar)`.

2. **Cobertura Preventiva:** Se detectó que el test `test_duplicados_de_proceso_eliminados` estaba viciado de origen
   (avalaba positivamente y obligaba borrar ambas filas). Se refactorizó la aserción para exigir que la salida retorne
   `len() == 2` conservando los vestigios únicos, pasando el escenario satisfactoriamente.

3. **Certificación QA Automática:** La rama arrojó 0 errores en Black (`-l 79`), 0 incidentes Flake8, y 100% de tipos
   avalados en `mypy`.

4. **Registro de Git:** El parche se empaquetó bajo la nueva normativa en la rama `qa/hotfix_duplicados_isin`.
