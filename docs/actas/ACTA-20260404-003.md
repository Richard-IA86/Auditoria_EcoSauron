## 3. Infraestructura de Corrección

### Herramienta principal

Script `/tmp/fix_docs_md2.py` (v2 con fix de bucle infinito):

- Wrapping de párrafos y listas a 120 caracteres (word-by-word)
- Eliminación de trailing spaces mediante `rstrip()`
- Inserción de blank lines alrededor de headings, fences y listas
- Pragmas de tabla: `<!-- pyml disable md013 -->`

### Descubrimiento crítico — sintaxis pragma

pymarkdown 0.9.36 requiere **una directiva por regla**:

```markdown
<!-- pyml disable md026 md036 -->  ← ERROR ('plugin md026 md036' not found)

<!-- pyml disable md026 -->        ← CORRECTO
<!-- pyml disable md036 -->        ← CORRECTO
```

### Config del pipeline (orquestador)

`config/markdownlint.json` del orquestador se aplica a TODOS los repos:

```json
{
  "plugins": {
    "md013": { "enabled": true, "line_length": 120,
               "code_blocks": false, "tables": false },
    "md033": { "enabled": false },
    "md041": { "enabled": false }
  }
}
```

`richard_ia86_dev` no tiene config propia — hereda del orquestador.

## 8. Conocimiento Adquirido

1. **Bug wrap infinito**: `rfind(" ", 0, 120)` falla con palabras
   sin espacios (URLs largas). Solución: acumulación word-by-word.

2. **Config orquestador global**: el pipeline no lee config local del
   repo auditado; siempre usa `auditoria_ecosauron/config/markdownlint.json`.

3. **Pragma una-regla-por-línea**: mandatorio en pymarkdown 0.9.36.

4. **MD029 + code blocks**: cambiar `2.` → `1.` en listas con fences
   embebidos genera cascada. Solución: pragma `disable/enable md029`.
