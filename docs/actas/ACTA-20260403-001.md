### 2.2 — Tarea 0b: Automatización gobierno de ramas

Se crearon dos scripts de apoyo al gobierno de ramas y se integró el
primero al pipeline de auditoría como PASO 0 no bloqueante.

#### `scripts/verificar_ramas.sh`
- Lee `config/repos.txt` y consulta la GitHub REST API con `curl + jq`
- Detecta ramas remotas sin PR (open, closed o mergeado) asociado
- Token resuelto en runtime desde `git credential store` (no en disco)
- Excluye: `main`, `copilot/*` (generadas automáticamente por Copilot)
- Salida: 0 = sin huérfanas | 1 = huérfanas detectadas

#### `scripts/cierre_jornada.sh`
- Bloque 1: orquestador — cambios sin commit + commits sin push
- Bloque 2: cada workspace — working tree + upstream sync
- Bloque 3: invoca `verificar_ramas.sh`
- Exit code = número de pendientes (0 = jornada limpia)

#### `run_audit.sh` — PASO 0 agregado
- No bloqueante: estado `RAMAS_OK` o `RAMAS_WARN` en tabla resumen
